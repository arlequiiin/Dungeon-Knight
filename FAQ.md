# Часто задаваемые вопросы (FAQ)

## Быстрые ответы на популярные вопросы

## 🎮 ИСПОЛЬЗОВАНИЕ

### Q1: Как запустить генератор?

**A:**
```
1. Меню → Dungeon → Setup → Create Default Tiles
2. Меню → Dungeon → Setup → Create Test Scene
3. Нажми Play ▶
```

Подробнее в [QUICK_START.md](QUICK_START.md)

---

### Q2: Где посмотреть созданные комнаты?

**A:**

**На экране (Game View):**
- Белые прямоугольники = комнаты
- Должны быть видны после нажатия Play

**В консоли (Console):**
```
[DungeonGenerator] Генерация завершена. Всего комнат: 35
```

**В Inspector (во время Play'а):**
- Выбери DungeonGenerator
- Смотри информацию о комнатах в компоненте

---

### Q3: Как изменить параметры генерации?

**A:**

Способ 1 (самый простой):
```
1. Выбери DungeonSettings.asset в Project
2. В Inspector измени параметры
3. Нажми Play - карта сгенерируется с новыми параметрами
```

Способ 2 (во время Play'а):
```
1. Выбери DungeonGenerator в иерархии
2. Нажми кнопку "Regenerate Dungeon"
3. Карта пересоздастся мгновенно
```

---

### Q4: Как всегда получать одну и ту же карту?

**A:**

Используй фиксированный seed:
```
DungeonSettings → Use Random Seed: отключи (галочка off)
DungeonSettings → Seed: введи 12345
```

Теперь каждый раз будет одна и та же карта - удобно для отладки.

---

### Q5: Как получать разные карты?

**A:**

Включи случайный seed:
```
DungeonSettings → Use Random Seed: включи (галочка on)
```

Каждый раз при Play будет новая карта.

## 📊 ПАРАМЕТРЫ

### Q6: Что означает "Branch Probability"?

**A:**

Это вероятность создания боковых веток (циклов в лабиринте).

- `0.0` = нет веток, линейный лабиринт
- `0.3` = небольшие боковые ветки (рекомендуется)
- `1.0` = максимум веток, сложный лабиринт

Пример:
```
0.0:  Start ─── Room ─── Room ─── Boss
                (линейный)

0.3:  Start ─── Room ──┐
           │           Room ─── Boss
           └─────────┘
          (с ветками)

1.0:  Start ─ Room ─ Room ─ Room
       │      │ │      │ │   │
       └──────┴─┴──────┴─┘   Boss
      (очень ветвистый)
```

---

### Q7: Что означает "Min/Max Path To Boss"?

**A:**

Это минимальное и максимальное расстояние от старта до босса.

- `minPathToBoss = 5` - минимум 5 "прыжков" до босса
- `maxPathToBoss = 10` - максимум 10 "прыжков"

Пример:
```
Min = 5:  Start → Room1 → Room2 → Room3 → Room4 → Room5 → Boss
          (5 переходов минимум)

Max = 10: Start → Room1 → Room2 → Room3 → Room4 → Room5 → ... → Boss
          (но не больше 10 переходов)
```

---

```

Normal Rooms Count: 5    ← Уменьши
```

## 🔍 ОТЛАДКА

### Q10: Я вижу только чёрный экран

**A:**

Возможные причины и решения:

1. **Камера смотрит в неправильное место:**
   ```
   - Выбери MainCamera
   - Установи позицию (50, 50, -10)
   - Orthographic Size: 50
   ```

2. **Tilemap скрыт:**
   ```
   - Выбери DungeonTilemap в иерархии
   - Убедись что не скрыт (иконка глаза)
   ```

3. **Нет Floor Tile:**
   ```
   - Выбери DungeonTilemap
   - В компоненте DungeonRenderer назначь Floor Tile
   ```

### Q11: Консоль показывает ошибку при запуске

**A:**

Проверь консоль на красные сообщения:

**Ошибка: "The type or namespace name 'DungeonConfig' could not be found"**
```
Решение:
1. Убедись что все файлы в папке Assets/Scrpits/Generator/
2. Жди пока Unity переcompилирует (может занять 30 сек)
3. Если не помогло - перезагрузи Unity
```

**Ошибка: "DungeonRenderer не найден на сцене"**
```
Решение:
1. Проверь что компонент DungeonRenderer добавлен на DungeonTilemap
2. В компоненте Generator назначь Dungeon Renderer вручную
```

## 🏗️ АРХИТЕКТУРА

### Q13: Как работает BSP разбиение?

**A:**

Рекурсивно:
```
1. Начинаем с полной карты
2. Делим по вертикали/горизонтали посередине
3. Для каждой половины повторяем процесс
4. Останавливаемся когда размер < минимума
5. В каждом "листе" создаём комнату
```

Результат: идеально сбалансированное бинарное дерево комнат.

Подробнее в [ARCHITECTURE_DECISION.md](ARCHITECTURE_DECISION.md)

---

### Q14: Как устроены типы комнат?

**A:**

```csharp
RoomType enum:
├─ Start   (1 комната) - стартовая, [0]
├─ Normal  (N комнат)  - с врагами
├─ Special (M комнат)  - магазины, ловушки
└─ Boss    (1 комната) - финальный враг, [последняя]
```

Где N = `normalRoomsCount`, M = `specialRoomsCount`

## 🚀 ПРОИЗВОДИТЕЛЬНОСТЬ

### Q16: Какой размер карты выдержит?

**A:**

**Без проблем:**
- До 500x500 тайлов
- До 500+ комнат
- Глубина BSP до 8 уровней

**Может быть медленнее:**
- Более 1000x1000 тайлов
- Более 1000 комнат

**Оптимизация:**
```csharp
// Уменьши максимальную глубину BSP
if (depth > 6) return;  // Было 8, теперь 6

// Уменьши количество комнат
config.normalRoomsCount = 20;  // Было 15
```

---

### Q17: Генерация слишком медленная

**A:**

Возможные причины:

1. **Слишком большая карта:**
   ```
   Уменьши mapWidth/mapHeight
   ```

2. **Слишком много комнат:**
   ```
   Уменьши normalRoomsCount
   ```

3. **Debug Mode включён:**
   ```
   Выключи Debug Mode в Generator
   (он выводит много информации в консоль)
   ```

4. **Слишком глубокое BSP:**
   ```csharp
   // Максимальная глубина ограничена:
   if (depth > 8) return;
   // Это нормально для большинства случаев
   ```
