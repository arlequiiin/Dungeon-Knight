## ⚠️ Если что-то не работает

### Ошибка: "The type or namespace name 'DungeonConfig' could not be found"

**Решение:**
- Проверь, что все файлы .cs находятся в папке `Assets/Scrpits/Generator/`
- Жди пока Unity переcompилирует проект (может занять 10-30 секунд)
- Если ошибка осталась - пересоздай файлы вручную

### Ошибка: "DungeonRenderer не найден на сцене"

**Решение:**
1. Убедись, что добавил компонент DungeonRenderer на GameObject с Tilemap
2. В компоненте Generator назначь поле "Dungeon Renderer" вручную

### Ошибка: Комнаты не видны на сцене

**Решение:**
1. Проверь, что Tilemap виден (добавлен в иерархию, не скрыт)
2. Проверь, что присвоены тайлы (FloorTile и WallTile)
3. Если тайлов нет - используй "Create Default Tiles" из меню

### Нет логов в консоли

**Решение:**
1. Убедись, что "Debug Mode" включён в компоненте Generator
2. Проверь что Play в режиме Play, а не Pause
3. Window → General → Console открывает консоль

### Консоль пустая после Play

**Решение:**
1. Проверь, что "Generate On Start" включён в Generator
2. Посмотри в Console на ошибки (красные сообщения)
3. Нажми Stop и заново нажми Play

## 🔄 Экспериментирование

### Изменение параметров генерации

```
1. Выбери DungeonSettings в Project (Assets/Resources/)
2. В Inspector измени параметры:
   - Map Width/Height - размер карты
   - normalRoomsCount - количество обычных комнат
   - minRoomSize/maxRoomSize - размеры комнат
   - и т.д.
3. Нажми Play
4. Подземелье сгенерируется с новыми параметрами
```

### Пересоздание подземелья во время Play'а

```
1. Во время Play'а выбери "DungeonGenerator" в иерархии
2. В Inspector нажми кнопку "Regenerate Dungeon"
3. Подземелье мгновенно пересоздастся
```

### Просмотр информации о комнатах

```
1. Во время Play'а посмотри в Console
2. Там выведен список всех комнат с координатами и типами
3. Используй эту информацию для отладки
```

## 🎓 Что делает генератор

1. **Разбивает карту** на прямоугольные регионы (BSP)
2. **Создаёт комнаты** в каждом регионе со случайными размерами
3. **Соединяет комнаты** логически (основа для коридоров)
4. **Распределяет типы** (старт, обычные, специальные, босс)
5. **Рисует на Tilemap** для визуализации

**Результат:** Полностью процедурная карта подземелья

## ❓ Часто задаваемые вопросы

**Q: Почему комнаты одного цвета?**
A: Потому что это минимальная версия. В будущем будут разные тайлы для разных типов.

**Q: Почему нет коридоров?**
A: Они в TODO. Сейчас только логическая связь комнат (основа для коридоров).

**Q: Можно ли менять параметры?**
A: Да! В DungeonConfig можно менять всё: размеры, количество, вероятность веток.

**Q: Где враги?**
A: Их спаун нужно добавить в отдельном скрипте. Генератор только создаёт геометрию.

**Q: Как добавить свои тайлы?**
A: Создай Tile Assets и присвой их в DungeonRenderer (FloorTile, WallTile).

**Q: Можно ли генерировать во время игры?**
A: Да! Просто вызови `generator.GenerateDungeon()` в любой момент.
