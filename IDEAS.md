# Dungeon Knight - Game Ideas

## Генерация 

1. Клеточный автомат для формы комнат

Применять только к Normal-комнатам (Start, Shop, Treasure оставить прямоугольными)
Убрать Boss комнату. Вместо неё когда игрок доходит до последней комнаты - появляется телепорт в комнату с боссом.
Алгоритм: заполнить прямоугольник комнаты случайным шумом (≈45% пола), прогнать 4–5 итераций правила «если у тайла ≥5 соседей-пола из 8 — он остаётся полом, иначе удаляется»
После автомата — flood fill от центра комнаты, удалить все островки пола, не связанные с основной областью (чтобы не было недостижимых зон)
Убрать текущий ModifyRoomShapes (CutCorner/AddProtrusion) или оставить как fallback для комнат, которые не прошли через автомат
Добавить в GridWalkConfig параметры: cellularFillChance (0.45), cellularIterations (4), cellularChance (вероятность что Normal-комната станет пещерной, например 0.5 — чтобы были и обычные и пещерные)

2. Генерация препятствий внутри комнат (новый Tilemap-слой)

Добавить в сцену третий Tilemap — obstacleTilemap (order -1, поверх ground на -2)
Тайл препятствия — следующий "уровень" или "высота" локации (блокирует проход)
Препятствия должны иметь коллайдер и влиять на NavMesh (чтобы мобы их обходили)
Генерировать только в Normal-комнатах, с отступом ≥2 тайла от краёв комнаты и от дверных проёмов (входов коридоров)
Алгоритм размещения — дискретизация диска Пуассона: задаём минимальное расстояние между препятствиями (например 3 тайла), генерируем точки, в каждой точке ставим кластер из 1–3 тайлов препятствий
Количество препятствий зависит от площади комнаты (например, density = 0.05 — 5% тайлов пола становятся препятствиями)
Добавить в GridWalkConfig: obstacleDensity, obstacleMinDistance (для Пуассона)
Добавить в CellData поле obstacleTiles (HashSet<Vector2Int>) — чтобы при спавне мобов и лута проверять, что точка не занята
В GridWalkRenderer добавить отрисовку obstacleTilemap

3. Размещение мобов и лута через диск Пуассона

Заменить текущий случайный выбор точки в MobSpawner.GetRandomFloorPosition на дискретизацию Пуассона
Генерировать набор допустимых точек спавна для каждой комнаты: точки пола, не занятые препятствиями, с отступом от стен, с минимальным расстоянием между собой
Из этого набора брать позиции для мобов, лута, декораций
Можно вынести Пуассона в отдельный утилитарный класс PoissonDiscSampler, который принимает набор допустимых тайлов, minDistance и rng, и возвращает список точек — переиспользовать для препятствий и для мобов

4. Вспомогательные изменения

CellData — добавить obstacleTiles, возможно spawnPoints (результат Пуассона для мобов)
GridWalkConfig — добавить секцию параметров клеточного автомата и секцию параметров препятствий
GridWalkDungeonGenerator.GenerateDungeon() — добавить шаги: после Generate() и RenderDungeon() вызвать генерацию препятствий, затем перезапечь NavMesh, затем спавн мобов
Убедиться, что коридоры никогда не получают препятствия (проверка: тайл принадлежит cell.floorTiles, а не коридору)

## Combat System

3. Система щитов
У некоторых мобов (ArmoredSkeleton) и героев (Knight, Templar) есть щит. Щит блокирует атаки с фронтальной стороны.
Мобы со щитом полностью блокируют фронтальный урон, при этом удар по щиту отнимает poise (устойчивость)
Нужно зайти с другой стороны или сломать щит тяжёлыми ударами (у тяжёлых ударов больше урона по устойчивости)

4. Стагер и Poise система
У каждого моба и героя есть параметр poise (устойчивость). Атаки снимают poise. При 0 poise — стагер (оглушение).
Каждая атака имеет `staggerDamage` помимо HP-урона
Лёгкие атаки: мало stagger, тяжёлые: много
Стагер = 1-2 сек оглушения, моб не может атаковать и двигаться
Во время стагера удары наносят x1.5 урона
У боссов poise медленно восстанавливается
У обычных мобов poise не восстанавливается до конца стагера

5. ~~Energy — восстановление ударами + расходники~~ ✅ РЕАЛИЗОВАНО
Энергия тратится на Dodge и Abilities. Восстанавливается ударами по врагам (фиксированно per attack из HeroData).
Dodge тратит dodgeEnergyCost, Ability тратит ability1EnergyCost. Проверки на сервере.
TODO: зелья маны как лут из сокровищниц, восстановление части энергии при зачистке комнаты.

6. Knockback при ударе
При попадании моб отлетает в направлении атаки.
Каждая атака имеет `knockbackForce`
Направление = от атакующего к цели
Тяжёлые атаки = сильный knockback, лёгкие = слабый
Круговые атаки разбрасывают от центра
Мобы тоже могут откидывать игрока (тяжёлые удары)
Knockback взаимодействует со стенами (можно прижать к стене)

7. Hitstop (микро-пауза при попадании)
При попадании игра замирает на 2-3 кадра. Создаёт ощущение "веса" удара.
`Time.timeScale` кратковременно = 0 (или `animator.speed = 0`)
Длительность зависит от силы удара
Камера может слегка трястись (screen shake)

## Mobs — Undead Crypt биом

Уже существующие префабы: `SkeletonWarrior`, `SkeletonGreatsword`, `ArmoredSkeleton`, `SkeletonArcher`, `SkeletonOverlord`

1. SkeletonWarrior — базовый мили
- Мили, средняя скорость, 2 атаки (лево/право)
- Простая погоня, атака в зоне досягаемости

2. SkeletonGreatsword — тяжёлый мили
- Медленный, но сильный. Большой двуручный меч.
- Медленные тяжёлые удары с большим knockback
- Высокий poise (тяжело стагернуть)
- Sweep-атака (круговая) — бьёт вокруг себя
- Долгий recovery после атаки — окно для наказания

3. ArmoredSkeleton — танк со щитом
- Щит блокирует фронтальные атаки
- Нужно зайти со спины или сломать щит тяжёлыми ударами
- Средняя скорость, умеренный урон

4. SkeletonArcher — дальний бой
- Держит дистанцию, стреляет стрелами
- Отходит если игрок слишком близко
- Слабый в ближнем бою

5. SkeletonOverlord — босс
- Появляется после телепорта из последней комнаты локации, периодически призывает других скелетов
- Увеличенный скелет, много HP, высокий poise
- 2 типа атак: быстрый удар, тяжёлый удар

## Mob AI — улучшения поведения

1. Базовый класс MobAI + расширенный FSM
- Абстрактный `MobAI` — общая логика (патруль, чейз, детект, хитбоксы, флип)
- Каждый тип моба наследует и переопределяет поведение
- Новые состояния FSM: `HitReaction`, `Stagger`, `Recovery`, `Retreat`
- `HitReaction` — короткая пауза при получении урона (flinch)
- `Recovery` — окно после атаки, моб уязвим
- `Retreat` — отступление (для Archer, low HP мобов)

2. Combo-атаки мобов
- Анимации Attack01/02/03 уже есть 
- Случайный выбор атаки 
- Разные атаки = разный урон, скорость, размер хитбокса

3. Group AI — координация мобов
- Не толпиться — если 3+ мобов вокруг игрока, дальние ждут очереди
- Окружение — мобы заходят с разных сторон
- Aggro sharing — если один заметил игрока, ближайшие тоже агрятся
