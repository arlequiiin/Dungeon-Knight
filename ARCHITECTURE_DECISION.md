## Почему именно Граф + BSP?

### BSP (Binary Space Partition)

**Что делает:**
- Рекурсивно разбивает пространство на прямоугольные регионы
- Гарантирует что **комнаты не пересекаются**
- Создаёт логическую иерархию комнат

**Плюсы:**
- ✅ Эффективное использование пространства
- ✅ Нет пересечений (геометрия чистая)
- ✅ Легко балансировать размеры комнат
- ✅ Может использовать разные правила разбиения

**Когда использовать:**
- Классические подземелья (замки, пещеры)
- Когда нужна чистая геометрия
- Когда важна оптимизация пространства

### Граф (Graph)

**Что делает:**
- Соединяет комнаты в логическую структуру
- Определяет пути между комнатами
- Позволяет контролировать сложность пути до босса
- Поддерживает циклы (петли маршрутов)

**Плюсы:**
- ✅ Контроль маршрутов
- ✅ Гарантия достижимости всех комнат
- ✅ Возможность создания боковых веток
- ✅ Вычисление расстояний между комнатами

**Когда использовать:**
- Когда важна структура лабиринта
- Когда нужны ветвящиеся маршруты
- Когда требуется управление сложностью

### Граф + BSP вместе

Это **идеальная комбинация**:

1. **BSP** обеспечивает **физическую геометрию** (где комнаты расположены)
2. **Граф** обеспечивает **логическую структуру** (как они соединены)

```
BSP:          Граф:

┌────┬──────┐  Room[0] ─── Room[1]
│    │      │     │           │
│    │      │     └─ Room[2] ─┘
├────┼──────┤          │
│    │      │      Room[3]
└────┴──────┘
```

## Сравнение с другими подходами

### ❌ Только BSP (без графа)
```
Проблемы:
- Комнаты создаются, но как их соединять?
- Нет контроля над маршрутами
- Сложно гарантировать достижимость
```

### ❌ Только Граф (без BSP)
```
Проблемы:
- Нет физического представления
- Трудно разместить комнаты без пересечений
- Нужна сложная система позиционирования
```

### ✅ Граф + BSP (текущий подход)
```
Преимущества:
- Чистая геометрия (BSP)
- Логическая структура (Граф)
- Лучшее из двух миров!
```
